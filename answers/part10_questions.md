## Вопрос 1

Что выведет этот код? Объясните порядок выполнения блоков.

```java
class Demo {
    static { System.out.print("A "); }
    { System.out.print("B "); }
    Demo() { System.out.print("C "); }
    public static void main(String[] args) {
        System.out.print("D ");
        new Demo();
        System.out.print("E ");
        new Demo();
    }
}
```

**Предсказанный вывод:**

```
A D B C E B C
```

**Объяснение:**

Статический блок выполняется один раз при загрузке класса: печатает "A ".

main печатает "D ".

первый new Demo():

блок инициализации экземпляра печатает "B "

конструктор печатает "C "

main печатает "E ".

второй new Demo() снова печатает "B C".

## Вопрос 2

Класс `Child` в пакете `b`, класс `Parent` в пакете `a`. `Child extends Parent`. Какие члены `Parent` доступны в `Child`? А какие доступны в другом классе пакета `b`, который НЕ наследует `Parent`?

**Ваш ответ:**

В Child (наследник из другого пакета) доступны: public и protected члены.
В другом классе пакета b (не наследнике) доступны только: public члены.
default (package-private) и private члены недоступны в обоих случаях, так как пакеты разные.

## Вопрос 4

Что выведет этот код? Объясните, что такое String Pool и как работает `intern()`.

```java
String a = "Hello";
String b = new String("Hello");
String c = b.intern();
System.out.println(a == b);
System.out.println(a == c);
System.out.println(b == c);
```

**Предсказанный вывод:**

```
false
true
false
```

**Объяснение:**

String Pool — область памяти для хранения уникальных строковых литералов.

a ссылается на объект в пуле.

b создает новый объект в куче (new String), поэтому a == b ложно.

b.intern() возвращает ссылку на объект из пула (так как "Hello" уже есть там благодаря a). Поэтому c ссылается на тот же объект, что и a.

a == c истинно.

b == c ложно, так как b в куче, а c в пуле.

## Вопрос 6

В чём разница между `StringBuilder` и `StringBuffer`? Приведите конкретный сценарий, когда нужен именно `StringBuffer`.

**Ваш ответ:**

StringBuilder не синхронизирован (быстрее, не потокобезопасен). StringBuffer синхронизирован (медленнее, потокобезопасен).

Сценарий для StringBuffer: Множественные потоки одновременно изменяют одну и ту же строку

## Вопрос 7

Что выведет этот код? Объясните, что происходит с переменной `x` в лямбде.

```java
int x = 10;
Runnable r = () -> System.out.println(x);
// x = 20;  // что будет, если раскомментировать?
r.run();
```

**Предсказанный вывод:**

```
10
```

**Объяснение:**

Лямбда-выражения могут захватывать только локальные переменные, которые являются final

Если раскомментировать x = 20;, код не скомпилируется

## Вопрос 8

Напишите пример лямбда-выражения, которое нельзя заменить на ссылку на метод. Объясните почему.

**Ваш пример и объяснение:**
Function<String, String> f = s -> s + "!";

Ссылка на метод подразумевает вызов существующего метода объекта или класса. Операция конкатенации со строковым литералом (+ "!") не является вызовом одного конкретного метода интерфейса Function, это составное выражение.

## Вопрос 9

Чем `EnumMap` быстрее `HashMap` для ключей-enum? Опишите внутреннюю реализацию в одном предложении.

**Ваш ответ:**

EnumMap использует простой массив, индексом которого служит порядковый номер константы enum (ordinal()), что обеспечивает доступ за O(1) без вычисления хеш-кода и коллизий.

## Вопрос 10

Дан `record Point(int x, int y) {}`. Что сгенерирует компилятор автоматически? Можно ли добавить в record изменяемое поле? Можно ли наследоваться от record?

**Ваш ответ:**

Генерирует: Приватные финальные поля, публичный конструктор со всеми аргументами, геттеры с именами полей, методы equals(), hashCode(), toString().

Изменяемое поле: Добавление обычного экземпляра поля, не описанного в заголовке record, запрещено.

Наследование: От record наследоваться нельзя
